# 通义听悟 (TingWu) 浏览器自动化操控技能

本技能用于通过浏览器自动化操控阿里云通义听悟平台，完成音视频转写、记录管理、智能总结查看、导出等全部操作。

4. **文本定位优先**：由于 CSS 类名会随版本更新变化，优先通过文本内容定位元素
5. **操作后验证**：每次关键操作后重新 snapshot 确认结果

## 操作流程

### 0. 前置检查：确认登录状态

**每次执行任务前，必须先确认用户已登录通义听悟。**

```
1. browser_navigate → https://tingwu.aliyun.com/home
2. browser_snapshot → 检查页面状态
```

### 1. 上传文件转写

**触发示例**：
- "帮我把这个文件转写一下"
- "上传录音到通义听悟"
- "转写这个视频文件"

**流程**：

```
步骤1: 进入上传页面
  - 方式A: 首页 → 点击 "上传音视频" 按钮
  - 方式B: 侧边栏 → "我的记录" → 点击 "新建" 按钮

步骤2: 选择上传方式
  - 点击 "上传本地文件" 或 "导入阿里云盘文件"

步骤3: 上传文件
  - 本地文件: 通过文件选择对话框上传
  - 阿里云盘: 浏览并选择文件

步骤4: 配置转写参数（snapshot 后根据页面选项操作）
  - 转写语言: 选择 "中文"/"英语"/"日语"/"粤语"/"中英文自由说"
  - 翻译功能: 可选开启
  - 区分发言人: 可选开启
  - 发言人数: 可选择人数

步骤5: 开始转写
  - 点击 "开始转写" 按钮
  - 等待转写完成（轮询检查状态）
```

**支持的文件格式**：
- 视频: mp4, wmv, m4v, flv, rmvb, dat, mov, mkv, webm, avi, mpeg, 3gp, ogg
- 音频: mp3, wav, m4a, wma, aac, ogg, amr, flac, aiff
- 视频限制: 单个 ≤ 6GB, 最长 6小时
- 音频限制: 单个 ≤ 500MB

## 通用操作模式

### 标准操作循环

```
1. browser_snapshot → 获取当前页面结构和元素引用
2. 分析 snapshot 结果，定位目标元素
3. browser_click / browser_type / browser_select_option → 执行操作
4. browser_wait_for → 等待页面响应
5. browser_snapshot → 验证操作结果
6. 如有需要，重复步骤2-5
```

### 等待策略

通义听悟是 SPA 应用，页面切换不会触发完整刷新，使用以下等待策略：

```
- 导航后: browser_wait_for text="目标页面特征文本" (如 "我的记录"、"转写原文")
- 弹窗出现: browser_wait_for text="弹窗标题" (如 "导出")
- 列表加载: browser_wait_for time=2 (固定等待2秒)
- 转写完成: browser_wait_for text="100%" 或检查进度条消失
```

### 错误处理

```
- 元素未找到 → 重新 snapshot，尝试通过不同文本定位
- 操作无响应 → 等待2秒后重新 snapshot 检查状态
- 页面跳转异常 → 重新导航到目标页面
- 登录过期 → 提示用户重新登录
```

## 注意事项

1. **登录状态**: 通义听悟使用阿里云账号登录，登录可能过期，需定期检查
2. **转写等待**: 大文件转写可能需要较长时间，需耐心等待并轮询状态
3. **文件上传**: 文件上传通过系统文件对话框，可能需要用户手动选择文件
4. **下载文件**: 导出下载的文件保存在浏览器默认下载目录
5. **页面动态性**: 页面内容是动态加载的，操作间需要适当等待
6. **iframe 注意**: 登录弹窗使用 iframe，但登录后操作均在主页面内，无需切换 iframe
